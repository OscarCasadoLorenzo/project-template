# Getting Started

This guide will help you set up the Project Template monorepo on your local machine.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** >= 22.11.0 LTS (we recommend using [nvm](https://github.com/nvm-sh/nvm))
- **pnpm** >= 9.0.0 (install via `npm install -g pnpm` or use corepack)
- **PostgreSQL** >= 14 (for the backend database)
- **Git** for version control

## Installation

### 1. Clone the Repository

```bash
git clone https://github.com/OscarCasadoLorenzo/project-template.git
cd project-template
```

### 2. Install Dependencies

The monorepo uses npm workspaces. Install all dependencies with:

```bash
npm install
```

This will install dependencies for all apps and packages in the monorepo.

### 3. Set Up Environment Variables

#### Frontend

Copy the example environment file:

```bash
cp apps/frontend/.env.example apps/frontend/.env.local
```

Configure the following variables:

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

#### Backend

Copy the example environment file:

```bash
cp apps/backend-rest/.env.example apps/backend-rest/.env
```

Configure the database connection and other variables:

```env
DATABASE_URL="postgresql://user:password@localhost:5432/projectdb?schema=public"
JWT_SECRET=your-secret-key-here
NODE_ENV=development
```

### 4. Set Up the Database

Navigate to the backend directory and run Prisma migrations:

```bash
cd apps/backend-rest
npx prisma migrate dev
npx prisma generate
cd ../..
```

### 5. Start Development Servers

You can start all apps simultaneously:

```bash
npm run dev
```

Or start individual apps:

```bash
# Frontend only (port 3000)
npm run dev --filter=@project-template/frontend

# Backend only (port 3001)
npm run dev --filter=@project-template/backend-rest

# Documentation (port 3002)
npm run dev --filter=@project-template/docs
```

## Verify Installation

Once the servers are running:

- **Frontend**: Open [http://localhost:3000](http://localhost:3000)
- **Backend API**: Open [http://localhost:3001/api](http://localhost:3001/api)
- **Documentation**: Open [http://localhost:3002](http://localhost:3002)

## Troubleshooting

### Port Already in Use

If you get a "port already in use" error, you can:

1. Kill the process using the port
2. Change the port in the respective app's `package.json`

### Database Connection Issues

Ensure PostgreSQL is running:

```bash
# macOS (Homebrew)
brew services start postgresql

# Linux (systemd)
sudo systemctl start postgresql
```

### Prisma Issues

If you encounter Prisma-related errors, try:

```bash
cd apps/backend-rest
npx prisma generate
npx prisma migrate reset --force
```
