# Toast Notification System

## Overview

The global toast notification system provides a consistent, accessible way to display user feedback across the application using **Sonner** (shadcn UI toast component) and **Jotai** for state management.

## Architecture

### Structure

The toast system is organized following our monorepo structure:

- **Frontend App (`apps/frontend`)**:
  - `/lib/atoms/toast.atoms.ts` - Jotai atoms for global state
  - `/hooks/useGlobalToast.ts` - Hook for triggering toasts
  - `/components/ToastProvider.tsx` - Provider component that renders toasts

- **UI Package (`packages/ui`)**:
  - `/src/primitives/toaster.tsx` - Sonner wrapper component (presentational only)

### Design Principles

- **Separation of Concerns**: UI package contains only presentational components; all business logic resides in the frontend app
- **Global State**: Jotai atoms enable toast triggering from anywhere in the app
- **Type Safety**: Full TypeScript support with well-defined interfaces
- **Accessibility**: Built on Sonner's accessible toast implementation

## Quick Example

```tsx
import { useGlobalToast } from "@/hooks";

function MyComponent() {
  const toast = useGlobalToast();

  return (
    <button onClick={() => toast.success("Operation completed!")}>
      Click me
    </button>
  );
}
```
