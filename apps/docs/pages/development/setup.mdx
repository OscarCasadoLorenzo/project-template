# Development Setup

Complete guide to setting up your development environment for the Project Template monorepo.

## System Requirements

### Required Software

- **Node.js**: Version 18.x or higher
- **npm**: Version 9.x or higher
- **Git**: Latest version
- **PostgreSQL**: Version 14 or higher
- **Code Editor**: VS Code (recommended)

### Recommended Tools

- **nvm**: Node version manager
- **Docker**: For containerized PostgreSQL
- **Postman**: API testing
- **GitHub Desktop**: Git GUI (optional)

## Installation

### 1. Install Node.js

Using nvm (recommended):

```bash
# Install nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash

# Install Node.js
nvm install 18
nvm use 18

# Verify installation
node --version  # Should show v18.x.x
npm --version   # Should show 9.x.x
```

### 2. Install PostgreSQL

**macOS (Homebrew)**:

```bash
brew install postgresql@14
brew services start postgresql@14
```

**Ubuntu/Debian**:

```bash
sudo apt update
sudo apt install postgresql-14
sudo systemctl start postgresql
```

**Windows**:
Download installer from [postgresql.org](https://www.postgresql.org/download/windows/)

**Docker (All platforms)**:

```bash
docker run --name postgres \
  -e POSTGRES_PASSWORD=password \
  -e POSTGRES_DB=projectdb \
  -p 5432:5432 \
  -d postgres:14
```

### 3. Clone Repository

```bash
git clone https://github.com/OscarCasadoLorenzo/project-template.git
cd project-template
```

### 4. Install Dependencies

```bash
npm install
```

This installs all dependencies for all apps and packages.

## Project Configuration

### Environment Variables

Create environment files for each app:

**Frontend** (`apps/frontend/.env.local`):

```env
NEXT_PUBLIC_API_URL=http://localhost:3001
```

**Backend** (`apps/backend-rest/.env`):

```env
DATABASE_URL="postgresql://postgres:password@localhost:5432/projectdb?schema=public"
JWT_SECRET=your-super-secret-jwt-key-change-in-production
JWT_EXPIRES_IN=7d
NODE_ENV=development
PORT=3001
```

### Database Setup

```bash
# Navigate to backend
cd apps/backend-rest

# Generate Prisma client
npx prisma generate

# Run migrations
npx prisma migrate dev

# (Optional) Seed database
npx prisma db seed

# Back to root
cd ../..
```

## VS Code Setup

### Recommended Extensions

Install these VS Code extensions:

```json
{
  "recommendations": [
    "dbaeumer.vscode-eslint",
    "esbenp.prettier-vscode",
    "bradlc.vscode-tailwindcss",
    "prisma.prisma",
    "github.copilot",
    "ms-vscode.vscode-typescript-next",
    "yoavbls.pretty-ts-errors"
  ]
}
```

### Workspace Settings

Create `.vscode/settings.json`:

```json
{
  "editor.defaultFormatter": "esbenp.prettier-vscode",
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "typescript.tsdk": "node_modules/typescript/lib",
  "typescript.enablePromptUseWorkspaceTsdk": true
}
```

## Running the Project

### Start All Apps

```bash
npm run dev
```

This starts:

- Frontend on [http://localhost:3000](http://localhost:3000)
- Backend on [http://localhost:3001](http://localhost:3001)
- Docs on [http://localhost:3002](http://localhost:3002)

### Start Individual Apps

```bash
# Frontend only
npm run dev --filter=@project-template/frontend

# Backend only
npm run dev --filter=@project-template/backend-rest

# Docs only
npm run dev --filter=@project-template/docs
```

## Development Workflow

### 1. Create Feature Branch

```bash
git checkout -b feature/my-new-feature
```

### 2. Make Changes

Edit files in your editor. Hot reload will update automatically.

### 3. Run Tests

```bash
# Run all tests
npm run test

# Run tests for specific app
npm run test --filter=@project-template/frontend
```

### 4. Lint Code

```bash
# Lint all code
npm run lint

# Fix linting issues
npm run lint --fix
```

### 5. Commit Changes

```bash
git add .
git commit -m "feat: add new feature"
```

Follow [Conventional Commits](https://www.conventionalcommits.org/) format.

### 6. Push and Create PR

```bash
git push origin feature/my-new-feature
```

Then create a Pull Request on GitHub.

## Troubleshooting

### Port Already in Use

Kill process on port:

```bash
# Find process
lsof -ti:3000

# Kill process
kill -9 <PID>
```

Or change port in `package.json`.

### Database Connection Failed

Check PostgreSQL is running:

```bash
# macOS
brew services list

# Linux
sudo systemctl status postgresql

# Docker
docker ps
```

### Module Not Found

Reinstall dependencies:

```bash
rm -rf node_modules
npm install
```

### Prisma Issues

Regenerate Prisma client:

```bash
cd apps/backend-rest
npx prisma generate
npx prisma migrate reset --force
```
